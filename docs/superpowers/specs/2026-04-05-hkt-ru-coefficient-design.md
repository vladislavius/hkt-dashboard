# Дизайн: уточнение коэффициента доли Пхукета в турпотоке РФ → Таиланд

**Дата:** 2026-04-05  
**Текущий коэффициент:** 0.38 (экспертная оценка)  
**Цель:** получить обоснованный коэффициент с погрешностью ±5%

---

## Контекст

В `collector.py` транзитные пассажиры из РФ считаются по формуле:
```
TAT_месяц × K × (1/30.44) × дней − прямые_пакс_RU = транзит
```
Где `K = 0.38` — доля Пхукета от общего турпотока РФ в Таиланд. Цифра взята как экспертная оценка без источника.

---

## Фаза 1 — Ручное исследование (прямо сейчас)

### Источники для поиска

| Источник | URL / раздел | Что ищем |
|---|---|---|
| TAT Intelligence Center | `mots.go.th` → Tourist Statistics → By Province | Россияне по провинции Пхукет vs всего Таиланд |
| BOT (Bank of Thailand) | Ежеквартальные отчёты → Tourism | Турпоток по аэропортам |
| AoT (Airports of Thailand) | Годовые отчёты → HKT passenger stats | Пассажиры по гражданству через HKT |

### Формула расчёта

```
K = TAT_Phuket_RU ÷ TAT_Thailand_RU
```

Проверить за минимум 4 разных месяца (разные сезоны), взять среднее. Ожидаемый диапазон: 0.30–0.45.

### Критерий успеха

Коэффициент подтверждён минимум двумя независимыми источниками за 2023-2024 год.

---

## Фаза 2 — Динамический расчёт (через 2-3 месяца)

После накопления истории прямых рейсов из РФ (Москва, Новосибирск и др.):

### Автоматический пересчёт в collector.py

```python
# Каждый месяц после накопления данных:
K_dynamic = прямые_пакс_RU_HKT / TAT_Phuket_RU
# Записывается в tat_stats.json как актуальный коэффициент
```

### Что меняется в архитектуре

- `tat_stats.json` получает поле `"coefficient_history"` — словарь `{месяц: K}`
- `collector.py` при расчёте транзита берёт `K` за соответствующий месяц, а не константу
- При отсутствии данных — fallback на среднее за последние 6 месяцев

### Критерий успеха

Коэффициент обновляется автоматически ежемесячно. История коэффициентов хранится и видна для аудита.
